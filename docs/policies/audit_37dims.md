# 37 维审计套件 · 承重该查什么（配 rubric.md 用）

inkos 37 维**适配到"压缩单源→60章"**（砍掉同人/跨书/正典隔离等不适用维度）。按 `rubric.md` 四轴归类。
实现于 `src/hiki/audit.py`（`DIMENSIONS` 注册表）+ `ledger.py`（时序）。

## 检测类型（决定可信度）

- **det 确定性结构检** —— LLM 裁判的**盲区**（人工校准实证：连 Opus 都漏掉场景重复/阵营串线），**最高价值、可信**。靠结构化 canon + 逐场景 delta，不走 LLM judge。
- **mech 机械正则** —— 可信、不可刷分。
- **llm LLM 判断** —— craft 维度。人工校准证明 **LLM 打"绝对分"不可靠**（高估 9-17），但**挑"毛病"(定性)与人工吻合**，故只作 **advisory 低权**。
- **metric 结构指标** —— 走 5 个可量化结构指标（爽点间隔/钩子密度等）。

## 用法

- **承重(det)**：plan 阶段 shift-left，硬伤(重复/串线/崩坏/翻转)触发重规划；伏笔序仅 advisory。
- **笔力(mech)**：成品上跑正则，命中即标。
- **人/故事性(llm)**：一次 craft 审计调用，**只挑毛病不打分**，advisory 低权。
- **关键纪律**：质量认证靠 det+mech+**人工 ground truth**，**绝不用 LLM 打绝对分自证**（Goodhart，连 Opus 都高估 9-17）。
