## 1. 目标与约束（全部已锁定）

| 项 | 定值 |
|---|---|
| 任务 | 任意章数/任意质量中文网文（48–85 万字）→ **固定 60 章 / ~21 万字** 复写 |
| 质量目标 | **稳定人工 95**（评分者=**网文读者**，95=网文标准顶级好看，**非文学性**） |
| 模型 | 仅 **deepseek-v4**：`v4-pro`($0.435/$0.87) + `v4-flash`($0.14/$0.28)，**1M 上下文**，缓存近免费。无 Claude/GPT/Gemini |
| 人工 | **生产链路零人工**；人=老师（事后逐本打分点评驱动学习） |
| 成本 | 单本硬上限 **¥50**（自适应；评估是成本大头） |
| 拒收 | **≤50%**；救不动/不可压缩的源拒收，绝不发砸招牌的货 |
| 原创性 | 保故事内核，**换皮+重写表达，禁逐字照搬**（避版权+去 AI 检测） |
| 其他 | 尽量高并发压周期；顶级泛化（各类题材） |

**输入现实**：
- **40 本标杆样本人工分 = 90**（高档锚定/参考集；既是 S/A 级参考水库种子，又是闸门校准的近目标锚——与金标 95 共同夹住 90↔95 边界）。
- **1万本生产输入仍鱼龙混杂**（弱源照样 C/D/Q）；脏数据多（防盗章/广告/作者的话/乱码/多书拼接）。
- **杠杆变小、风险大降**：强源(≈90)→95 只需 **+5**（高保真压缩+轻润+选择性锦标赛），远易于早期估的 +20；弱源(10k 中)仍需大幅升级或拒收。

## 2. 设计公理（一切由此推导——优雅与可维护之根）

> 整套系统不是 N 个独立机制的堆叠，而是 8 条公理的推论。新增任何东西前，先问"它属于哪条公理"。

- **A1 文件即状态，阶段即纯函数**：阶段 = `输入文件 → 输出文件`，不持运行时状态。⇒ 可独立 rerun / 断点续 / 单测 / 故障隔离。
- **A2 场景是原子单元**：质量在场景粒度发生；锦标赛、精修、缓存、评估全部 per-scene。⇒ 小单元 = 高质量可达 + 天然并行。
- **A3 一个质量环**：`生成 → 评估 → 保留/升级/拒收`。全系统只有这一个环，由(模式, 预算)参数化。⇒ 概念最小化。
- **A4 源是脊柱；提分靠"选"不靠"写"**：保住源已验证的钩子/爽点；提分主要来自 BoN/锦标赛吃模型尾部，而非让模型"写得更好"。强源(90)只需小幅提分(+5)，弱源靠模式升级。
- **A5 评估靠对照不靠绝对**：永不用 DeepSeek 绝对自评当分（必虚高）。对照——上锚=外部金标(95)、近目标锚=40 本标杆(90)、下锚=本书源(须明显胜过)、活体人工真值。⇒ 破 Goodhart。
- **A6 凡影响产出者皆版本化资产**：prompt/闸门/金标/检测器/偏差模型/配置 全部 pinned+versioned。⇒ 可复现、回归可归因、可回滚。
- **A7 成本随难度自适应；救不动则拒**：易书少烧、难书猛烧、触顶即拒。⇒ 可扩展 + 兑现 95 承诺。
- **A8 人是老师不是操作员**：生产在线全自动；学习离线异步。⇒ 零人工生产 + 数据飞轮，二者解耦。

## 3. 核心抽象（5 个概念，正交）

```
①管线 Stages (A1)      ②场景单元 Scene (A2)
   ingest→extract→plan→draft→evaluate→assemble
        │                    │
        └──────── ③质量环 Loop (A3,A4,A7) ────────┐
                   生成→评估→保留/升级/拒收         │
                            │                       │
                   ④闸门 Gate (A5)  ←──────────────┘
                  对照(金标95/标杆90/源) + 偏差校准 + 整本闸门
                            ▲
                   ⑤飞轮 Flywheel (A6,A8) 离线
                  人工点评 → 校准器/检测器/金标/(微调)
   ──────────────────────────────────────────────
   状态 = 文件 (A1)：per-book artifacts + global versioned assets
```

## 4. 管线：6 个纯阶段（A1）

每阶段 `输入文件 → 输出文件`，schema 校验输出（坏 LLM 输出直接拒，不落盘）。

| 阶段 | 输入 | 输出 | 模型档 | 要点 |
|---|---|---|---|---|
| **0 Ingest** | 源 .txt | `clean.txt` + `meta` | flash | **一等清洗管线**：去防盗章/广告/作者的话、编码修复、章节完整性、去重、**多书拼接边界检测**。万本真翻车点在此 |
| **1 Extract** | clean.txt | `dna/*`（脊柱/钩子账/情感曲线/人物弧/伏笔/爽点/人名词典/语域指纹/题材） | pro | 1M 上下文可大块/整本入栈，**缓存源**；超 80 万字才需分块 |
| **2 Plan** | dna | `outline_60` + `scenes[]`（每场景:源指针+DRAMATIZE/SUMMARIZE+beat） | pro | **分层规划**：全书弧→卷→章→场景 + 一致性回环；自带"压缩保真"校验（高价值钩子/爽点保留率） |
| **3 Draft** | scenes + dna(缓存) | `scene/*.md` | flash 主力 | 场景为单元；prompt 哲学="提纯源"非"重写"；走质量环(§5) |
| **4 Evaluate** | scene/book | `gate/*` | pro/flash | 闸门(§6)；场景级 + 整本级 |
| **5 Assemble** | 通过的 scene | `final.md` + `acceptance` | — | 拼装 + 验收报告(认证/成本/风险) |

**准入分级**（Ingest 后、付费前）：质量等级 **S/A/B/C/D/Q** × **可压缩性**（form-incompatible 的纯刷数值/累积流→Q）。等级 → 质量环入口模式。标杆 40 本≈S/A(90)。

## 5. 质量环：唯一的控制循环（A3/A4/A7）

**模式 = 升级阶梯的级**（转化深度递增）；准入等级定入口，闸门不过则下沉一级，最深仍不过即拒收。

```
入口(由S/A/B/C/D定)
 模式1 保真压缩   删五类注水+修语病+人名归一+场景化(杀流水账)   ← 强源(90)主路径,只需+5
 模式2 强化改写   补强弱钩子/立体人物/重铺节奏 + 关键章 BoN
 模式3 类型化重构 按题材范式重排beat + 参考水库注入
 模式4 概念级重启 仅保概念+大幅重建 + 参考水库注入（最高风险，达不到95→并入Q）
   每级内，每个场景跑：
     生成 N 候选(多样化) →[A4] Elo成对锦标赛选优 → 对最弱维度精修 → 复评
     loop until 维度达标 或 触场景预算
   造峰：全书 3–5 戏剧高点给最大 N/最多轮
```

- **选择性锦标赛(A7)**：大 N 仅投戏剧高点 + 闸门判出的最弱场景；其余小 N。**评估比生成贵**，N 与 PK 次数按 ¥50 自适应。
- **引擎即模式内的动作**（不再单列）：场景化=模式1基操；网文级精修(强化网文读者买单维度,非文学辞藻)=BoN 候选的生成指令；宏观架构=Plan 阶段；锦标赛=环本身。

---

## 6. 闸门：评估（A5，破 Goodhart）

**三类信号**（机械主导，绝不用绝对自评）：
1. **机械层(0 LLM，一票否决前置)**：AI腔密度 / 风格cosine vs 源 / 注水残留 / 语病残留+人名一致性 / 结构合规(60章·~3500字·4-beat·首章交代) / **逐字重合检测(原创性下限)**。
2. **对照成对 PK(A5)**：每场景须 **明显胜过本书源同类场景(下锚)** 且 **达上锚(外部金标神作片段，同题材同功能，95)**；40 本标杆(90)作近目标校准锚帮助定 90↔95 边界。去偏：随机序+双向取平均+多判取中+传递性校验。
3. **整本级闸门**：弧光收束 / 节奏曲线 / 主题统一 / 结尾落点 / **全书连续性审计**（人物状态机/伏笔账/时间线）。60 个 95 章 ≠ 一本 95 书。

**判定**：机械过 + 场景对照过 + 整本过 → 认证 95。**冷启动**靠金标(95)+标杆(90)锚；**稳态**由偏差校准器(§7)把认证对齐真人工 95。
**reader panel**：同模型盲点相关 → 仅作补充、低权，不当独立真值。

## 7. 学习飞轮：离线，破 Goodhart 主干（A6/A8）

**人=老师**：生产全自动出货后，人工**逐本打分点评**（M0–M3 深度长评；M4 快评+抽样）。点评进 `hfl.jsonl`（人工总分+维度分+自由点评+当时自动信号快照+**版本号**）。

驱动三件事：
- **偏差校准器**：学 `预测人工分=f(自动信号)`，把"自动虚高"显式拟合掉 → 校准闸门到真 95。
- **维度偏差图 + 点评挖掘**：找出自动高估的维度 → 重加权 + 自动扩检测器（数据驱动版 ANTI-SLOP）。
- **金标/参考库扩充**：高分场景反哺（金标固定防漂移，参考库滚动扩；40 本标杆 90 是初始种子）。

**泛化原则（数据≠越多越好）**：泛化 ∝ 覆盖度×新鲜度×均衡度×标注质量 ÷ 冗余，**不∝数量**。故：
- **主动采样**：优先评边界书/新题材/auto-金标分歧大/**抽样被拒书**(防选择性偏差)，而非堆量。
- **覆盖度仪表盘**：按题材/质量档/失效模式盯空白，导向人评力气。
- **版本时间衰减**：旧版本标签降权（防移动靶污染）。
- **泛化 KPI = 留出题材/留出源 表现**（非训练拟合）。
- **新题材**：通用失效(AI腔/流水账/语病)迁移好；题材特异质量不迁移，须补该题材种子锚本。

**在线学习纪律**：批量定期重训 + holdout 验证 + 版本钉死，不每本即时改（防抖动）。
**能力天花板**：数据无法把闸门/生成抬过 DeepSeek 判别力上限——**突破只能微调**（§11 路线 K）。

## 8. 可靠 / 可扩展 / 可维护（工程公理落地）

**Reliable（A1/A6）**
- 文件即状态；阶段纯函数、幂等、可独立 rerun；全量断点续跑。
- schema 校验所有 LLM 输出，坏数据拒收不落盘（学 inkos Zod+immutable delta）。
- 锦标赛非确定 → **候选/PK 结果缓存 + 种子化**，崩溃不重烧、可复现。
- 单本预算硬上限 ¥50，跑飞自动 abort 转拒收；**单本失败隔离**，不拖累全批。
- 优雅降级：LLM 失败退避重试 → 仍失败转隔离队列。

**Scalable（A2/A7/A8）**
- **两级并发**：inter-book 工作队列 + 无状态 worker 池；intra-book 场景扇出。
- **缓存源上下文**为省钱/提速命脉（同源被 60 章×锦标赛×评估反复复用）。
- 限流感知客户端 + 指数退避；成本随难度自适应（非一刀切）。
- 学习飞轮异步解耦于生产，互不阻塞。

**Maintainable（A6）**
- **凡影响产出者皆版本化资产**：`assets/{prompts,gate,gold_corpus,ref_reservoir,detectors,bias_model,config}` 全 pinned；任一升级对金标库做**回归测试**（不得退化）。
- **配置驱动**：模式/预算/阈值/路由在 config，不在代码。
- **阶段间契约 = schema**；prompt 为数据模板非硬编码。
- 可观测内建：通过率/拒收率/false-reject/认证↔人工误差/PK胜率/成本分布/覆盖度/漂移告警。

## 10. 成本模型（单本硬上限 ¥50；1 元≈$0.14）

| 环节 | 估算 | 说明 |
|---|---|---|
| Extract | ~$0.2 | 源 1M 首读 flash，缓存后近免费 |
| Plan | ~$0.3 | DNA 入栈 + pro 迭代 |
| Draft(1 份) | ~$0.26 | 300k out × pro，缓存源 |
| 选择性锦标赛+精修 | ~$0.7–1.2 | 高点/弱场景大 N |
| **闸门(PK+panel)** | **~$2–3（大头）** | 评估比生成贵 |
| 合计 | **≈$4–5 ≈ ¥30–36** | 在 ¥50 内 |

关键：DeepSeek 极廉+1M+缓存使生成便宜，**绑死成本的是评估**；缓存源是命脉。强源(90)只需 +5、模式1 即可，成本偏下沿。M0 实测校正。

## 12. 关键不确定性（只能实测，诚实在先）

1. **强源 90→95 的 +5 自动拔高**（已大降级风险）：纯 DeepSeek、¥50、强源(标杆级)顶到 95——比早期 +20 估计可信得多。M0 先验；不成立则松旋钮。**真正的难度在弱源(10k 中)的大幅拔高**，靠模式升级 + 拒收兜底。
2. **【命门】闸门判别力**：DeepSeek 做 90↔95 区分有上限；40 本标杆(90)+金标(95)双锚 + 人工逐本打分持续校准缓解，天花板靠微调突破。
3. **压缩 vs 网文爽感**：4:1 压缩可能伤养成/追更快感 → 可压缩性维度兜底(不可压缩即 Q)。
4. **学习速度 < 生成速度**：放量前置闸防裸奔出万本未校准货。
5. **微调路线(K)**：DeepSeek 无官方微调 API；V4 开源权重(MIT)可自托管微调——飞轮数据足够后评估 ROI，是突破判别力天花板 + 压成本的后期最大杠杆，但重运维、非快赢。
