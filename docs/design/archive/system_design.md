# hiki-fiction-rewrite · 小说复写系统设计 spec（v3 重构版）

**版本：** v3.0（基于 40 本语料全普查 + 1万本鱼龙混杂规模需求重构）
**日期：** 2026-06-07
**与 system_design_old.md 的关系：** old 是 PR-11 时代"从零参考源创作"架构，人工分卡在 60–70。本文档是**推倒重来的复写/压缩架构**，目标人工 85–95，纯 DeepSeek-v4、全自动、规模化（1万本+）。

---

## 0. TL;DR（一页读懂）

- **任务**：把任意章数、**任意质量**（40 本标杆样本人工分=75）的中文网文（48–85 万字为主）压缩复写成**固定 60 章 / ~21 万字**、**稳定人工评分 95** 的成品。
- **硬约束**：唯一模型族 deepseek-v4（reasoner/chat/flash，无 Claude/GPT/Gemini）｜**生产链路零人工**（人工仅事后逐本打分点评当"老师"驱动学习，§7.3）｜单本成本 **≤¥50**｜拒收率 **≤50%**｜尽量高并发压周期｜顶级泛化。
- **核心哲学**：**分层保真 + 自适应升级 + 加法拔高**。减法（保钩子/删水/修病）封顶于源（~75→83-85）；冲 95 必须加法——**95 靠"选"不靠"写"**（场景级 BoN + Elo 锦标赛吃模型尾部）+ **逐场景穷尽托底**消方差（§16）。
- **规模落地四支柱**：① 源准入 S/A/B/C/D/Q 分级（花钱前体检，Q 拒收）② 生产模式=升级阶梯（保真压缩/强化改写/类型化重构/概念级重启）③ **双向对照闸门**（无 95 标注 → 不做绝对打分，改"明显胜过 75 分下锚 + 达外部金标上锚"成对 PK）④ 规模化工程 + 数据飞轮。
- **"稳定 95"的真实兑现**：冷启动闸门用外部金标库当上锚；稳态由**人工逐本打分**持续校准闸门（§7.3，把"自动虚高"显式拟合掉）。只发货过 95 线的书，达不到即拒（≤50%）。**全系统单点风险 = 纯 DeepSeek 能否把 75 顶到 95**，M0 第一优先验证；不成立则松一个旋钮。

## 1. 为什么推倒重来（旧系统根因 + 语料实证）

### 1.1 旧系统卡在 60–70 的根因

旧系统自评 93–96 / Claude 70–79 / **人工 65**，自动分与人工分差 28–31 分——它在**优化一个和人类判断不相关的代理指标**，且靠往 Coase prompt 堆 15 条硬规则（9000+ 字符）已到边际收益递减。人工挑出的所有毛病本质都是**"偏离源"**：

| 人工反馈 | 真因 |
|---|---|
| 第一章不交代主角/背景 | 压缩丢了铺垫 |
| **文笔像文艺不像网文** | DeepSeek 漂成"文艺 AI 腔"，丢了源语域 |
| 对话短句蹦字 | DeepSeek 对话默认 |
| godmode / 重生信息差解冲突 | 模型自作主张"优化"剧情、偏离源逻辑 |
| 反派纸片 / 配角工具人 | 压缩删了源的人物刻画 beat |

**结论**：旧系统把任务当成"参考源重新创作"，用 DeepSeek 通用文笔覆盖了源已被验证的好。新系统反过来——**源是脊柱，我们删水、修病、轻润，而非重写。**

### 1.2 40 本语料全普查的实证结论（驱动每一条设计）

普查方法：8 并行 agent × 5 本，每本开头 + 中部 + 结尾抽样，行号引证。关键发现：

1. **全是完整长篇（48–85 万字，1 本约 16 万字偏短）** → 任务确认是纯压缩（2.5:1 ~ 4:1），不存在扩写。
   - ⚠️ 工程教训：普查中部分 agent 因未正确取全长（Read 默认前 2000 行）误判"残卷"。**生产系统 P0 必须对全长文件稳健分块**，否则会犯"只读到前 10%"的同类错误。
2. **高度同构** → 一套泛化管线即可：`开局极端钩子（受辱/绝境/背叛/退婚/绝症/炮灰命）→ 金手指（系统/空间/重生/穿越/穿书/异能/预知）→ 单主角第三人称限知 → 爽点循环（信息差降维碾压+打脸+逐级升级+收后宫/甜宠）→ 大团圆生子收尾`。
3. **语域全是网文白话**（口语、对话/吐槽驱动、短句、题材梗+网络梗、感叹号密集、单句拆行）。"漂文艺腔失分"实锤。
4. **注水形态可机械识别**（压缩金矿），五类：① 逐招打斗复述 ② 心理/震惊独白堆叠（"不可能/为什么"刷屏）③ 单句拆行排版 ④ 群众围观/弹幕舆论戏车轱辘 ⑤ 信息倾倒式设定/商业流水账。
5. **语言瑕疵成系统**：错别字（"尔虞我诈"→"尔你我诈"、"毁于一旦"→"毁为一旦"）、人名前后不一（赵苒苒/赵冉冉、商幽/商明珠）、国名不一（少元国/邵元国）、输入法连带错字（"反社"→"报社"反复）。→ **修语病 + 统一术语是确定性提分项**，DeepSeek 能可靠做、可对照验证、不 Goodhart。
6. **新增防御**：发现疑似"文件尾部拼接了另一部小说"（主角突变）→ P0 必须做主线边界/异常检测；人物改名残留普遍 → 需全程人名归一化。

### 1.3 规模需求带来的质的变化

40 本是**人工删选高分**；1万本是**鱼龙混杂各类**（已确认）。这打破"源即质量"前提——大量源本身只有 50–70 分，纯高保真压缩只能得 50–70。因此系统必须**按源质量自适应**：强源压缩、弱源升级、无救源拒收（拒收已确认可接受）。

## 2. 核心设计哲学

### 2.1 分层保真（系统灵魂）

| 层 | 策略 | 为什么 |
|---|---|---|
| **故事层** | **高保真（保 + 夯实）**：钩子、爽点循环、人物动机、情感 beat、金手指设定 | 这是源高分的真正来源；被注水稀释的爽点要重新夯实 |
| **语言层** | **保语域 + 净化**：锁网文白话/对话驱动/短句/题材梗（不漂文艺），同时删五类注水、修系统性语病、统一人名术语 | 同时干掉旧系统两大失分项；净化是 DeepSeek 可靠且不 Goodhart 的稳定提分项 |

### 2.2 自适应保真：源书分级 → 生产模式（应对鱼龙混杂）

源准入给每本打 **S/A/B/C/D/Q** 六级，分级决定**生产模式**（转化深度逐级加大）：

| 源书等级 | 生产模式 | 行为 | 转化深度 / 风险 |
|---|---|---|---|
| **S / A** | 保真压缩 | 进入生产 | 最低：保故事+保语域，删水/修病/轻润 |
| **B** | 强化改写 | 进入生产 | 中低：保主线骨架，补强弱钩子/立体人物/重铺节奏 |
| **C** | 类型化重构 | 进入生产 | 中高：DNA 不齐，按题材成熟范式重组钩子/节奏，重排增删 beat |
| **D** | 概念级重启 | 进入生产，**允许大幅改动** | 最高：仅保概念/金手指，情节人物大幅重建 |
| **Q** | 不可救 / 不可生产 | **拒绝开始任务，不生成 60 章** | — |

**关键工程判断（必须正视）**：转化越深，**越偏离源已被验证的质量**——而"源是质量脊柱"正是纯 DeepSeek 能够到 85–95 的根本杠杆。**D=概念级重启 ≈ 从零创作 = 旧系统封顶 65–70 的失败区**。所以：

- **C/D 不能靠"弱源 + DeepSeek 默认"硬生**（必漂文艺腔/godmode、回旧系统老路）。
- **解法：参考语料注入（Reference-Corpus Injection）**。把 S/A 级语料按题材沉淀为"质量水库"——成熟的钩子布局、爽点节奏模板、语域 few-shot。C/D 生产时 = **弱源概念 + 同题材强源的结构/语域模板** 重组。质量来自**强源水库**而非 DeepSeek 凭空，"源是质量"原则对弱源依旧成立。
- D 档是全系统**最难兑现 85–95 的路径**，MVP 阶段优先验证 S/A/B/C；D 先小样本验证"参考注入后能否稳定 ≥85"，达不到则把 D 并入 Q（拒收）。

### 2.3 破 Goodhart 三原则（无 Claude 的命门）

1. **真值锚定**：评"相对源丢了多少"（损失，可验证），不评"绝对多好"（易刷分）。
2. **机械优先**：正则/统计信号无法被模型刷分，主导闸门。
3. **对照优先**：DeepSeek 判"A vs B/源 谁更优"远比"给 A 打绝对分"可靠。
4. **绝不把 DeepSeek 绝对自评当优化目标或质量保证**（旧系统的坑，必虚高 ~25 分）。

### 2.4 决策状态

**已确认（用户拍板，不再变动）：**
- ✅ 输入为 1万本鱼龙混杂各类源（40 本标杆样本人工分 = 75）；**允许拒收**，拒收率上限 **50%**。
- ✅ **目标：稳定输出 95 分**（出货每本过双向对照 95 线，达不到即拒）。
- ✅ **评分者 = 网文读者，"95" = 网文读者标准下的顶级好看**（爽点/钩子/代入/情感冲击/节奏 为主，**非文学性**）→ 引擎 B 是"网文级精修"非"文学拔高"（§16.3）。需先立 5–10 本"这就是 95"锚本 + 评分 rubric（§18-A）。
- ✅ **原创性下限**：保故事内核与爽点，但**换皮 + 重写表达**——人名/专名换新、具体设定细节改写、**禁逐字照搬源**（超 N 字连续重合即判抄）。既避抄袭/版权风险，又顺带去 AI 检测（§18-I）。
- ✅ **生产链路零人工**（生成/闸门/拒收全自动）；**人工 = 老师**：事后**逐本打分点评**驱动持续偏差学习（§7.3），含**抽样点评被拒书**防选择性偏差。评分强度自适应（M0–M3 深度长评 → M4 快评+抽样）。
- ✅ **单本成本上限 ¥50**（选择性锦标赛，自适应算力）。
- ✅ **外部金标范例库**作为闸门上锚（一次性静态资产、不进生产链路、不算人工干预）。
- ✅ 源书分级 **S/A/B/C/D/Q → 生产模式**（保真压缩/强化改写/类型化重构/概念级重启/拒收）。
- ✅ **参考语料水库注入**用于 C/D 模式（质量来自 S/A 强源水库，非 DeepSeek 凭空）。
- ✅ **D 档达不到稳定目标即降级为拒收（并入 Q）**——绝不强发压不到线的成品。

**默认假设（可改）：**
- **A｜保真目的**：自适应保真，产出"同一故事的更高质量版本"，不强制换名换设定（若需差异化/规避雷同，在 P2 加"差异化映射层"，质量风险中等）。
- **B｜规格**：固定 60 章 / ~21 万字，每章 ~3500 字，压缩比随源浮动。
- **C｜成本**：模式即阶梯，S/A ≈¥5–8、D ≈¥30–40，单本硬上限 ≈¥40 后拒收/隔离。

## 4. 支柱 A · 源准入分级（Admission Control）

**目的**：花大钱前先判断"这本能否被自动提升到 ≥85"，把救不动的源挡在门外（保护承诺、省钱）。

**输入**：源全文（仅抽样读：开头 + 等距中段若干 + 结尾，约占全文 5–10%）。

**DNA 体检维度**（deepseek-v4-reasoner，输出结构化 JSON）：
- **故事强度**（正向）：开局钩子强度、爽点循环密度、人物动机清晰度、情感 payoff、利害逐级升级、有无完整收束。
- **缺陷负载**（可修）：注水率（五类形态占比估计）、语病率、人名/设定一致性、是否疑似多书拼接。

**输出分级（S/A/B/C/D/Q）→ 生产模式**：
| 等级 | 判据 | 生产模式 | 预估天花板 |
|---|---|---|---|
| **S** | DNA 极强、缺陷少（标杆级） | 保真压缩 | 92–95 |
| **A** | DNA 强、缺陷以"可修"为主 | 保真压缩 | 88–93 |
| **B** | DNA 尚可、有明显弱点（弱钩子/扁人物/节奏注水） | 强化改写 | 86–91 |
| **C** | DNA 不齐（亮点零散、结构松散） | 类型化重构（+参考语料注入） | 85–89 |
| **D** | DNA 弱、仅概念可用 | 概念级重启（+参考语料注入，允许大幅改动） | 85–88（最难，风险最高） |
| **Q** | 概念都无亮点 / 不可救 / 疑似脏数据无法清洗 | **拒收，不生成 60 章** | — |

- 分级阈值**离线用 40 本已知人工分校准**（与闸门同一套校准机制，见支柱 C）。
- **正交维度：可压缩性（§18-C）**。质量分级（S–D/Q）之外，独立判"该源能否经受 4:1 压缩"——纯数值系统流/无主线种田流"魅力即刷/累积"，压缩后成空壳，即便质量不低也应判**form-incompatible → Q 或特殊处理**。准入 = 质量等级 × 可压缩性。
- **产物**：`admission.json`（质量等级 + 可压缩性 + 各维度分 + 预估天花板 + 生产模式 + 拒收理由 + 建议参考语料桶）。Q 级源不进入后续付费阶段。

## 5. 核心管线 P0–P6

### P0 · 摄入 & 切分
- 解析任意 .txt（UTF-8/GBK 自动嗅探），**全长稳健分块**（不依赖单次 Read/单次上下文；DeepSeek 上下文仅 64–128K，无法整本入栈）。
- 章节边界识别（`第X章` 等正则 + 兜底按字数切块）。
- **异常检测**：主角/视角突变、风格指纹突跳 → 疑似多书拼接 → 标记主线边界或拒收。
- 建源 RAG 索引（向量库，供 P3 章节锚定检索；embedding 用 deepseek 兼容或本地）。

### P1 · 故事 DNA 提取（map-reduce，**高并发**）
- **map（并行）**：源切 N 段，每段并行抽取 events / hooks / 人物 beat / 情感 beat / 代价 / 金句 / 意象 / 设定。
- **reduce**：合并为
  - `story_spine.md` 主线脊柱 + 卷结构
  - `hook_ledger.md` 钩子账本（位置/类型/强度）
  - `emotion_curve.md` 情感曲线
  - `character_arcs.md` 人物弧光（动机/转变/代价）
  - `foreshadowing.md` 伏笔账本（plant→payoff）
  - `payoff_list.md` 爽点清单
  - `name_dict.md` **人名/术语归一词典**（修复"赵苒苒/赵冉冉"类不一致，全程强制）
  - `voice_fingerprint.json` 语域指纹（句长分布/对话比/短句率/感叹号率/题材梗库 + few-shot 范例段）
  - `genre.json` 题材识别（只用于挑语域基调，不套情节公式 → 保泛化）

### P2 · 60 章压缩映射
- 把脊柱映射到**恰好 60 章**：选择哪些 beat 存活、为每章分配**源区间**（供 P3 RAG 锚定）。这是"保什么/砍什么"的压缩决策层——核心手艺。
- 每章产出：源区间指针 + 4-beat（opening_hook / conflict_escalation / midpoint_twist / ending_cliff）+ 本章钩子 + 涉及伏笔 plant/payoff + 人物状态。
- 一致性校验：伏笔 plant→payoff 完整、人物弧光连续、爽点均匀分布。
- 弱源在此处做**升级标注**（哪些钩子需补强、哪些人物需立体化）。
- 产物：`outline_60.md` + per-章 `chapter_brief`。

### P3 · 章节复写（**源锚定 → 60 章高并发**）
- **关键优势**：每章锚定具体源区间，章间依赖远小于从零生成 → **60 章可大规模并行起草**（这是压缩生产周期的结构性来源）。仅 ch1 / 卷边界取前章结尾做缝合。
- 单章注入：本章源区间原文（充足长度）+ voice_fingerprint 范例 + chapter_brief（钩子/4-beat）+ 相邻章摘要（非全文）+ 人物状态 + name_dict。
- **prompt 哲学**（区别于旧系统）："**把这段源压缩进 ~3500 字：保住下列钩子/情感 beat[来自DNA]，用源语域[来自指纹]，删掉五类注水，修掉语病，统一人名。**" —— 是**提纯**，不是重写。
- 模型：deepseek-v4-chat 主力起草（走量）；强源 ch1/卷首/全书末等关键章可 BoN 多候选。
- **出处**：场景单元=NovelForge 分场景 / libriscribe SCENE_PROMPT；定向修复=Long-Novel-GPT diff 局部修改；并行=Long-Novel-GPT 多线程。详见 §16 引擎 A/D、§17。

### P5 · 全局缝合 & 润色（P4 见支柱 C）
- 跨章连续性检查（伏笔回收、人物状态、时间线）。
- **跨 60 章疲劳/重复检测**（词汇疲劳、开头/结尾模式重复 → variance brief，借 inkos long-span-fatigue 思路）。
- 终轮去 AI 腔 scrub（机械黑名单 + 轻润）。

### P6 · 组装 + 验收报告
- 拼接 60 章 → `final.md`（目录 + 卷分隔）。
- `acceptance.json`：认证分（校准闸门）+ 各维度分 + 成本 + 升级到第几 Tier + 风险/残留问题。

## 6. 支柱 B · 生产模式 = 升级阶梯（统一模型）

**核心设计：4 种生产模式本身就是升级阶梯的 4 级。** 源准入等级决定**入口模式**；闸门不过则**下沉到更深一级模式**重做；4 级都不过则隔离/拒收。这把"按源分级的静态路由"和"按闸门失败的动态升级"统一成一套。

```
入口模式（由 S/A/B/C/D 决定起点）
   ↓
模式1 保真压缩      = 压缩起草 + 机械净化（删五类注水/修语病/人名归一）
   │                  ＋ 对照源定向修复最弱章（diff 式手术，非整章重写）
   │ 闸门 ≥阈值 → 通过 ;  <阈值 ↓
模式2 强化改写      = 保主线骨架，补强弱钩子 / 立体化反派配角 / 重铺注水节奏
   │                  ＋ 关键章 BoN 多候选
   │ <阈值 ↓
模式3 类型化重构    = 按题材成熟范式重排/增删 beat ＋【参考语料注入】同题材强源结构/语域模板
   │ <阈值 ↓
模式4 概念级重启    = 仅保概念/金手指，大幅重建 ＋【参考语料注入】（最高风险）
   │ <阈值（最深模式仍不达标）↓
隔离/拒收（事后 Q）  标记"无法自动认证"，不发货
```

- **入口即起点，不必每本从模式1 爬起**：S/A 直接进模式1；B 进模式2；C 进模式3；D 进模式4（D 已是最深，闸门不过即隔离）。这样既省钱（强源不做无谓重构）又托底（弱源直接上重武器）。
- **成本**：S/A ≈¥5–8；B ≈¥12–18；C ≈¥20–30；D ≈¥30–40；单本硬上限 ≈¥40，超限转隔离。
- **修复/重构触发项来自闸门的具体损失清单**（哪章丢了哪个钩子、哪个人物扁、哪段漂文艺腔），不是笼统"重写"。
- **参考语料注入（模式3/4 专属）**：从 S/A 级语料按题材沉淀的"质量水库"取成熟钩子布局 / 爽点节奏模板 / 语域 few-shot，喂给重构/重启——质量来自强源水库，不靠 DeepSeek 凭空（避免回旧系统老路）。

## 7. 支柱 C · 校准自动闸门（整个系统的命门）

**为什么是命门**：**全程零人工** + 无 Claude，则这个 DeepSeek 闸门**就是质量保证本身**。但唯一人工标注只有 **40 本 = 75 分（无任何 90/95 样本）**，且 DeepSeek 分不清 92 vs 95 → **闸门没有"95 上锚"，无法用绝对打分卡 95**。解法：**双向对照闸门**（见 7.2）。

### 7.1 闸门信号（三类，绝不用绝对自评当分）

**① 机械层（0 LLM，主导，不可刷分）**
- AI 腔密度（黑名单正则命中/千字）
- 风格 cosine vs 源 voice_fingerprint（漂移检测）
- **注水残留**（五类形态检测器：重复度/拆行率/车轱辘 n-gram）
- **语病残留**（错别字/成语误用词典 + 人名一致性 checker vs name_dict）
- 结构合规（60 章 / 每章 ~3500 字 / 4-beat 存在性 / 第一章 4 要素交代）

**② 对照式故事保真（LLM，pairwise，损失导向）**
- 每章 vs 源区间："**相对源，丢失/弱化了哪些钩子/情感/动机？加了哪些 AI 腔？**" → 结构化损失清单。
- 聚合为"故事保真度"（保留率，可验证），而非"好不好"。

**③ reader panel（LLM，多人格，默认挑刺）**
- N 个 DeepSeek 人格各管一个具体失效维度：第一章交代、爽点兑现、人物立体、对话真实、节奏。每人默认 refute，投票。

**认证 = 双向对照通过（不做绝对打分，见 7.2），机械层为一票否决前置门。**

**出处**：双免疫（机械正则 + LLM 裁判）= autonovel ANTI-SLOP；reader panel = autonovel reader_panel；37 维裁剪 = inkos 连续性审计；anti-detect 去AI腔 = inkos；成对 PK = autonovel compare_chapters Elo。⚠️ autonovel 用 Opus 当裁判，我们以 DeepSeek 多人格 + **金标范例对照** + 奖励模型替代——95 闸门最大风险点。详见 §17.1。

### 7.2 双向对照闸门（冷启动卡 95）+ 人工分持续校准（主干，见 7.3）

**两阶段定位**：
- **冷启动（无人工标注积累前）**：闸门靠**双锚成对对照**判定（下文）。这是 M0 起步时的唯一可信办法。
- **稳态（人工逐本打分积累后）**：闸门主校准信号转为**真·人工分**（§7.3），双向对照退为"生成期场景选优机制 + 防漂移校验"，外部金标库退为**防漂移稳定锚 + 生成范例**（不再是唯一上锚）。

**双锚成对对照**（绝对打分 → 双锚 PK，因冷启动无 95 标注）：

| 锚 | 来源 | 判据 |
|---|---|---|
| **下锚** | 40 本 = 75 分（已知人工分） | 成品场景须**明显胜过** 75 分源同类场景 |
| **上锚** | **外部金标范例库**（一次性整理的现成优秀已出版网文/文学片段，90–95 水准；**静态资产、不进生产链路、不算人工干预**） | 成品场景须**达到金标同级** |

```
每个场景认证：
  机械层一票否决（AI腔/注水/语病/结构 任一超限 → 直接打回）
  ② 成对 PK：明显胜过下锚（75分源同类场景）？  否 → 打回 Tier 升级
  ③ 成对 PK：达到上锚（金标范例同级）？        否 → 打回 Tier 升级
  两关皆过 → 该场景"够 95"
书认证 = 全 60 章所有场景皆过（最弱一环裁决，呼应 §16.1 的 p⁶⁰）
```

- **防飞轮漂移**：数据飞轮可扩范例库，但**外部金标上锚固定不动**，防止"闸门只认自己产物"的 Goodhart 回归（§16.4）。
- **40 本的真正用途**：当下锚 + 机械检测器/词典的训练语料（不是上锚——它们只有 75）。
- prompt/config/闸门/金标库版本化 → 质量回归可归因、可回滚。
- ⚠️ **残余风险**：纯 DeepSeek 做"达金标同级"PK 判别力有上限——但 §7.3 的人工逐本打分把它从"只能赌"变成"每本都在校正"。

### 7.3 人工反馈账本（HFL）+ 持续偏差学习（破 Goodhart 主干）

**定位**：旧系统的根本错误是"自动 93 / 人工 65"的偏差**始终隐性**（人读反馈手写硬规则打补丁）。新系统把它变成**数据闭环**——偏差 = 可测量、可学习、可自动修正。

**人工 = 老师，不是操作员**：生成/闸门判定/拒收全自动（生产链路零人工）；人只在**事后逐本打分点评**，系统据此学习。这与"生产零人工"不冲突。

**① 人工反馈账本（append-only 全局资产）**，每条记录：
- 人工总分 + 维度子分（钩子/语域/人物/对话/节奏…）
- **自由文本点评**（最值钱：记 WHY，如"第一章看不懂主角""对话像机器人""中段无聊"）
- **当时自动信号快照**（机械分 + 对照 PK + reader panel）→ 算残差 `人工 − 自动`
- **系统版本号**（prompt/闸门/金标/模型）→ 偏差归因到具体版本

**② 偏差修正模型（calibrator）**：学 `预测人工分 = f(自动信号)`，把闸门判断校正到"预测人工"。逐本打分天然覆盖全质量档 → 学出**真校准曲线**（非固定偏移），把"自动虚高"显式拟合掉。

**③ 维度偏差图**：算出哪些自动维度系统性高估 → 给闸门重加权、给被高估维度加强检测器。

**④ 点评挖掘 → 自动扩检测器/人格**：自由文本点评聚类成复发失效模式 → 自动生成新机械检测器 / reader-panel 人格 / 对照维度（= 数据驱动版 autonovel ANTI-SLOP，而非手写）。

**评分强度（自适应）**：M0–M3 低量期**逐本深度长评**（教学黄金期）；M4 规模期转**快速打分+短点评 + 高比例抽样**。

**防选择性偏差（必做）**：不只点评出货书，**同时抽样点评被拒书**，才能测 false-reject、防闸门越收越保守。

**在线学习纪律**：偏差模型**批量定期重训 + holdout 验证 + 版本钉死**，不每本即时改（防抖动/过拟合）；人工分滞后于出货，校正的是**未来书**。

## 8. 支柱 D · 规模化工程（1万本优雅运维）

- **两级并发**：① inter-book worker 池跨万本调度；② intra-book 章节扇出（P1 map / P3 起草天然并行）。总吞吐受 DeepSeek 限流 + 预算约束。
- **DeepSeek 客户端**：限流感知 + 指数退避 + 5xx 重试；并发信号量。
- **文件即状态 + 全量断点续跑**：中间产物全落盘 `output/<slug>/`，单 stage 可独立 rerun；中断自动从断点续（沿用旧系统已验证机制）。
- **故障隔离**：单本失败不拖累全批；失败入重试队列。
- **成本治理**：单本成本账本 + **单本硬上限 ¥50**（跑飞自动 abort 转隔离）；全局成本仪表盘。
- **可观测性**：通过率 / 拒收率（上限 50%）/ 成本分布 / 升级模式分布 / 对照 PK 胜率分布 / 漂移告警。
- **自动 QA（零人工）**：闸门通过率 / 拒收率 / PK 胜率分布的统计监控 + 对**固定金标库**的回归测试（闸门版本升级时，对金标库判定不得退化）。**无人工点检**（约束所限）。

## 9. 模型路由与并发（deepseek-v4 only）

| 阶段 | 档位 | 理由 |
|---|---|---|
| 源准入 DNA 体检 | reasoner | 需判断/推理 |
| P1 提取 / P2 映射 | reasoner | 结构化推理、压缩决策 |
| P3 章节起草 | chat（主力，走量）| 流畅性 + 成本 |
| 关键章 BoN | chat 多候选（高温）| 候选多样性 |
| P4 对照评估 / reader panel | reasoner | 评判质量 |
| 机械净化 / 检测器 | 0 LLM（正则/统计）+ flash 兜底 | 最省 |
| Tier1/2 修复 | chat / reasoner | 视干预深度 |

**实际模型（2026-06 核查）**：`deepseek-v4-flash`（$0.14 in / $0.28 out 每 M tokens）、`deepseek-v4-pro`（$0.435 / $0.87）；**两者上下文均 1M tokens**；缓存命中近乎免费（$0.0028/$0.0036 每 M）。`deepseek-chat/reasoner` 2026-07-24 停用 → 用 v4-flash（非思考=chat，思考=reasoner 档）+ v4-pro。本文"reasoner/chat/flash 档"= v4-pro / v4-flash思考 / v4-flash非思考。

> ✅ **1M 上下文是重大利好（修正早期 64–128K 误判）**：50–80 万字源大部分可整本/大块入栈，**P0 不必激进分块、P1 提取可大窗口、P3 可注入丰富源上下文 + 全量 DNA**，RAG 从"必需"降为"超长源(>80万字)的可选优化"。架构因此更接近 autonovel 的"1M 全量加载"路线，质量更高、工程更简。
> 💡 **缓存是成本命脉**：源上下文一次写入缓存，后续 60 章起草 + 锦标赛 + 评估反复复用近乎免费 → 必须用 prompt 缓存。

## 10. 成本模型（目标稳定 95；单本硬上限 ¥50）

**单本硬上限 ¥50（≈$6.9），超限 abort 转拒收。** 真实 token 预算（DeepSeek 实价 + 缓存，1 元≈$0.14）：

| 环节 | 估算 | 说明 |
|---|---|---|
| P1 提取 | ~$0.2 | 源 1M tokens 首读 $0.14（flash），缓存后近免费 |
| P2 分层规划 | ~$0.3 | DNA 入栈 + pro 迭代推理 |
| P3 起草（1 份全书） | ~$0.26 | 60×3500字≈300k out × pro $0.87/M（缓存源） |
| 选择性锦标赛 + 精修 | ~$0.7–1.2 | 高点/弱场景大 N，其余小 N |
| **闸门（PK + reader panel）** | **~$2–3（成本大头）** | 逐场景成对 PK 输入重；**评估比生成贵** |
| 合计 | **≈$4–5 ≈ ¥30–36** | 在 ¥50 内，但贴近上限 |

- **关键结论**：DeepSeek 极廉 + 1M + 缓存使生成很便宜，**绑死成本的是评估（PK/panel）**——锦标赛 N 与 PK 次数必须按预算自适应；缓存源上下文是省钱命脉。
- **选择性锦标赛**：大 N 仅投 3–5 戏剧高点 + 闸门判出最弱场景（§16.5）。
- ⚠️ 上表为粗估，**M0 必须实测校正**；若评估成本压不进 ¥50，则降 PK 密度 / 用 flash 做初筛 PK + pro 仅终裁，或松成本旋钮。

## 11. 泛化策略（应对"各类"）

- **结构模板无关**：P1 DNA 提取从源"长"出脊柱，不套爽文公式；题材识别只挑语域基调。
- **语域自动**：voice_fingerprint 每本现学，天然覆盖未见题材（悬疑/科幻/历史正剧等）。
- **机械资产可配**：注水黑名单 / 语病词典 / 题材梗库随语料增长持续扩充、版本化。
- **参考语料水库（全局资产）**：S/A 级源与成品按题材沉淀"质量水库"（钩子布局 / 爽点节奏模板 / 语域 few-shot），供 C/D 模式注入；随生产滚动扩充（S/A 成品反哺），是 C/D 升级的质量来源。
- **外部金标范例库（全局资产，闸门上锚）**：一次性整理的现成优秀已出版网文/文学片段（90–95 水准），按题材组织；作为双向对照闸门的**上锚**（§7.2），**固定不动**以防飞轮 Goodhart 漂移。静态资产、不进生产链路。与"参考水库"区别：水库供生成注入（可滚动扩），金标库供闸门判定（固定）。
- **闸门校准跨题材分层**：必要时按题材分桶校准阈值（不同题材人工评分基线不同）。

## 16. 质量引擎：从 85 → 95（quality elevation）

§1–§15 的"分层保真 + 净化"是**减法**（删水/修病/保钩子），封顶于源质量（源约 75–80）→ 成品约 **83–85**。冲 90、尤其稳定 95，必须叠加**加法（手艺拔高）+ 方差消除**。

### 16.1 为什么"稳定 95"是方差问题（决定架构形态）

人工按**最弱一环**评分。设单章独立达 95 的概率为 p，整本 60 章全 ≥95 ≈ **p⁶⁰**；要整本稳定 95 达 90%，需 p≈0.998。
→ **稳定 95 = 逐场景穷尽托底（零弱环）+ 选优造峰**，无捷径。

### 16.2 核心杠杆：95 靠"选"不靠"写"

DeepSeek 单次输出是质量分布（中位~80，P95~92）。改不了模型，就**吃尾部**：不让它写一个 95，而是写 N 个候选、最好的是 95，再可靠地选出来。问题化简为：① 候选够多样让尾部够到 95；② 选择器可靠到能认出 95（成对锦标赛 + 对照 95 范例，非绝对打分）。即**推理期算力换质量**，**在场景级做**（单元小→尾部易够到、成本低、天然并行）。

### 16.3 四个质量引擎（把减法变加法）

- **引擎 A · 场景化复写单元**（杀流水账）：P2 单元从"源区间→章"改为"选关键场景 DRAMATIZE（完整演出）+ 其余 SUMMARIZE（过渡）"。每章 = 1–2 个充分戏剧化场景 + 薄连接。出处：NovelForge 分场景、libriscribe SCENE_PROMPT。
- **引擎 B · 网文级精修 + 真范例锚定**（破封顶于源）：⚠️**已锁定评分者=网文读者（§2.4），故 B 不是"文学拔高"而是"网文级精修"**——加强网文读者真正买单的维度（钩子张力 / 爽点兑现力度 / 情感冲击 / 人物魅力与代入 / 对话鲜活 / 画面感），而非辞藻文学性（后者对网文读者反而扣分="水文学化"）。用**真正 95 级同题材神作片段 few-shot**（来自金标库），模仿"好看的网文"而非模型平均腔。把参考水库升级为"含场景级神作范例"。
- **引擎 C · 主动宏观架构**（比源更紧）：P2 主动设计 60 章戏剧架构（起承转合/中点反转/逐级 set-piece/高潮 payoff），用参考水库宏观模板重排源 beat，而非搬运松散连载结构。
- **引擎 D · 场景级锦标赛精修**（吃尾部）：每场景 N 候选→Elo 成对锦标赛选优→对最弱维度精修→**loop-until 所有维度 ≥ 目标或触预算**。全书 3–5 个戏剧高点给最大 N/最多轮次**造峰**。出处：autonovel 保留/丢弃 + compare_chapters Elo。

### 16.4 优雅可靠 = 数据飞轮（由人工逐本打分驱动）

**逐本打分点评（§7.3）是飞轮的显式信号**：每本人工分+点评 → ① 持续校准偏差模型/闸门（pass 越来越准等于人工 95）② 锐化维度权重 + 点评挖掘自动扩检测器 ③ 高分场景入题材范例库（生成下限越来越高）。量越大、下限越高、闸门越准、方差越小——**优雅可靠 = 随产量自增强的学习系统，非写死流水线。** 外部金标库作为固定上锚防飞轮 Goodhart 漂移。

### 16.5 三旋钮已定稿（用户拍板）

| 旋钮 | 定值 | 推导出的设计约束 |
|---|---|---|
| **成本上限** | **¥50/本** | = ¥50–150 区间下沿 → **选择性锦标赛**：大 N 只投全书 3–5 个戏剧高点 + 闸门判出的最弱场景，其余小 N。够用但紧，不能全书铺大 N |
| **人工** | **全程零人工**（生产+校准都不许） | 唯一标注 = 40 本×75 分，无 90/95 样本 → 闸门无绝对上锚 → **改双向对照闸门 + 外部金标库上锚**（§7.2，用户已接受） |
| **拒收率上限** | **50%** | 至少一半的 75 分源须被纯 DeepSeek 自动顶到 95（+20 拔高）——全系统最激进指标 |

- **目标**：稳定 95（出货每本过双向对照 95 线，达不到即拒，拒收 ≤50%）。
- ⚠️ **核心不确定性**：① 纯 DeepSeek 能否把 ≥50% 的 75 分源在 ¥50 内顶到 95；② 纯 DeepSeek 的"达金标同级"PK 判别力够不够。**两者只能 M0/M1 实测**；若顶不动，须松一个旋钮（提价 / 容更高拒收 / 降到稳定 90）。引擎 A–D（§16.3）为达标必需。

## 18. 缺口方向决策（v3.1，按重要度拍板）

对自审发现的 20 个缺口逐条定向。标注：🔴动摇目标级 / 🟠重大 / 🟡严谨性 / 🟢工程。

**🔴 A · "95"定义 + 评分者**（地基）：锁定**评分者 = 网文读者**，95 = 网文标准顶级好看，非文学性。**先做**：立 5–10 本"这就是 95"锚本 + 评分 rubric（钩子/爽点/代入/情感/节奏/对话/人物魅力 维度），作金标库种子与人工评分基准。→ 已入 §2.4 / §16.3。

**🔴 B · +20 点数预算**：拒绝"信仰式 +20"。M0 跑**消融实验**（各引擎单独开关测人工分增量），把 +20 拆成可归因预算；某引擎不达预期即砍。→ 入 §14 M0。

**🔴 C · 可压缩性正交维度**：质量之外独立判"题材-压缩兼容性"，魅力即刷/累积的纯系统流/无主线种田 → form-incompatible → Q。→ 已入 §4。

**🟠 D · P0 升级为一等清洗管线**：去防盗章/广告/作者的话 + 编码修复 + 章节完整性 + 去重 + 多书拼接边界检测，带质检指标。**万本规模真正翻车点在此，不在创作。**

**🟠 E · P2 分层规划协议**：全书弧 → 卷 → 章 → 场景**逐层细化 + 一致性回环**（1M 上下文使大窗口规划可行）；P2 自带闸门。每章 ~3500 字 + 爽点均布 + 伏笔不断 作为约束满足。

**🟠 F · P2 压缩保真评估**：独立于 prose 闸门，对照源 DNA 评"高价值钩子/爽点/情感峰保留率"——防"文笔好但砍错主线"。

**🟠 G · 整本级闸门**：60 个 95 章 ≠ 一本 95 书。增整本闸门（弧光收束/节奏曲线/主题统一/结尾落点/全书连续性审计），与场景闸门正交。

**🟠 H · 放量前置闸（学习速度 vs 生成速度）**：飞轮须先于规模收敛。硬门 = "偏差模型 pred↔人工误差 < 阈值 且 累计 N 本已评" 才解锁 M4，否则裸奔出万本未校准货。

**🟡 G-rigor · ML/评估纪律**（合并 9–13）：① PK 去偏（随机序+双向取平均+多判取中+传递性校验；金标须同题材同功能配对）② reader panel 同模型盲点相关→诚实降权，独立性靠机械+金标+人工 ③ 偏差模型协变量漂移→边界主动采样；生成**禁直接优化奖励模型分**（防二级 Goodhart，须金标+人工双锁）④ 40 本留 holdout，禁训练即测试 ⑤ 人工评分 rubric + 锚本 + 双评一致性抽查。

**🟢 I · 原创性下限**：换名/换专名/改设定细节/重写表达，禁逐字照搬（连续重合超 N 字判抄）→ 避版权 + 顺带去 AI 检测。已入 §2.4。

**🟢 J · 确定性与续跑**：锦标赛高温非确定 → 候选/PK 结果缓存 + 种子化，产物当不可变 artifact 落盘，崩溃不重烧。

**🔵 K · 微调（最大杠杆，但需自托管）**：DeepSeek **无官方微调 API**，V4 为开源权重(MIT) → 微调须**自托管/三方训练开源权重**（重运维）。**降级为后期可选**：飞轮数据足够后，评估"自托管微调一个复写模型(烤入网文语域、去 AI 腔) + 判分模型(提判别力)"的 ROI——可能同时冲 95 与压成本，但不是快赢。先 prompt+推理期跑通，数据攒够再评估。

**🔵 L · 便宜可靠性增益**：extraction/judging 自一致性多采样取共识；标题/简介生成（网文里第一印象分）纳入成品交付。
