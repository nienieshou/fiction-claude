# 项目终结结论(2026-06-12)· 权威收口

> 综合本轮全部实测(自迭代 R13c-R16 + 判别器实验 + 稳定75探索 M1/M3/M6)的最终结论。

## 一、能力画像:从困 63 到稳 71(+8 实质突破)

| 阶段 | 锚分(团宠/暴君) | 关键动作 |
|---|---|---|
| 自迭代 R13c-R16 | 困 **63**(打地鼠均衡) | 控制面/账本/高潮单场景化/macro剧情进展唯一——每轮治一类缺陷,总分不动 |
| 换判别器 + 承重工程 | 68(S源) | 修产线噪声判别器 bug(PICK→GOLD_PK) |
| **best-of-3 N=3** | **71**(S源零硬伤) | best-of-K 用确定性交付门挑零硬伤本,把 ±5 方差变优势 |

**工程把锚分从困死的 63 实质推进到稳定 71**,且 71 是**零硬伤、可商用、交付门零误放**的本。这是真实进展。

## 二、可兑现的交付配方

**best-of-3 N=3 / S源 / 选 ship_issues 最少本 / ¥22/本** → 稳定交付 **71-72**(零硬伤)。
- best-of-K(`scripts/best_of.py`)是核心:同源跑 K 次,用交付门硬伤计数选最优,绕开"整本质量判别器"难题。
- 交付门(R13c-R16 标定)零误放:9+本生产仅放行裸过门的本。

## 三、三个上限(为什么停在 71,75 需微调,90 不可达)

### 模型上限(最硬)
- 笔力(73-75)/人(70-73)是**生成侧能力**,确定性工程碰不到。
- **判别器死结**:对照 GOLD_PK 仅 75% 精度(实验实测),且区分度低时失灵——用 ~70 分判别力的模型选不出 90 分作品。
- **大N 尾部 ceiling**:N=12 选优生成侧 +1-2(实测 bok2 文笔确实更凝练),但被承重方差抵消,净≈0,成本翻倍(ROI负)。test-time compute 的尾部到不了 75。

### 设计上限
- 4:1 压缩 + 换皮重写 → 复写 < 源(负提纯)。生产源~75复写到63,损耗而非提纯。
- 60章固定 + 单源 → 注水拆章(邻章重演根因)。

### 题材上限
- 宫斗/虐恋(线性叙事)> 修仙(注水拆章):暴君S源71 > 团宠修仙63。
- 题材天花板差异 ~5-8 分。

## 四、目标可达性裁定

| 目标 | 可达性 | 路径 |
|---|---|---|
| **71-72**(S源零硬伤) | ✅ **工程已兑现** | best-of-3 N=3 |
| **73** | ⚠️ ROI 差 | best-of-5 N=12(¥90+) |
| **75** | ❌ 工程到顶 | **需微调**(生成侧最后3-4分,点修/大N都补不上) |
| **90** | ❌ 能力墙 | 需中文网文创作达人类优秀作者水平的模型(等基座进步) |

**点修对零硬伤本无效**(点修修硬伤);**大N ROI负**;**75→90 是人类级创作力,LLM 整体前沿**。

## 五、务实成功线重定义(替换 PRD"稳定95")

PRD 的"稳定95"和 v5"稳定90/75"在当前 DeepSeek+复写架构下均不可达。与工程能兑现的现实对齐:

```
成功线(新):
  S源     稳定交付 71-72(best-of-3,零硬伤可商用)
  顶级A源  稳定交付 68-70
  普通A源  良品线 63-67 或拒收
  拒收率   接受 >50%(选择性交付的代价)
  探75     仅靠微调(路线K)
```

## 六、微调(路线K)启动条件

唯一能破 71→75 的杠杆:
- **前置**:飞轮 HFL 从 39 条扩到 100+(主动采样边界本/留出题材/被拒书),覆盖各题材。
- **目标**:抬笔力/人两维(生成侧分布上移),DeepSeek-V4 开源权重(MIT)自托管微调。
- **ROI 判据**:微调后笔力/人提升带来的分增 vs 重运维成本。
- **预期**:逼近 75-78,仍难到 90。

## 七、最成熟的资产

迭代系统本身(自托管循环 + stop_table + 配对差分 + 引文 grep 验真 + best-of-K + 交付门零误放 + batch 台账)是项目**最成熟、可直接复用**的部分——服务运营态质量监控 + 微调飞轮。

## 八、下一步

1. **泛化验证**(进行中):5 本不同题材 best-of-3,验"系统能否在陌生题材稳定交付 71 零硬伤"——见 `batch_summary_generalization.md`。
2. **若泛化成立** → 转运营态:min_grade=A 选好源,best-of-3 量产,选择性交付 71-72。

## 九、泛化验证修正(2026-06-12,5本陌生题材 best-of-3)

**实测推翻"best-of-3 任意题材稳71零硬伤"**——5本(玄幻S/种田S/末世/盗墓/兽世)best-of-3,**裸过门仅1/5**(兽世养崽),零误放(末世Q正确拒收)。

| 结论 | 实测 |
|---|---|
| 工程机制泛化 | ✓成立(控制面/账本/交付门/准入分级5题材都work,零误放,Q拒收正确,不过拟合) |
| 交付质量泛化 | ⚠️弱+强题材依赖(裸过1/5,best-of-3对多线/升级流不够) |
| 版本互斥/重演 | 跨题材头号残留(盗墓5处最重),多线题材复发,B2再确认 |

**修正后可兑现配方(分题材)**:
- 线性叙事(宫斗/虐恋/兽世/种田): best-of-3~5 → 稳68-71零硬伤
- 多线/升级流(玄幻/盗墓/修仙): best-of-5+ 或良品线(硬伤分≤1),版本互斥头号
- form-incompatible(末世数值流): Q拒收

**题材适配是Fable分的强预测变量**。运营态按题材天花板差异化(详见 output/batch_summary_generalization.md)。

**一句话**:工程做到了它能做的极限(63→71,零硬伤可商用);再上一层(75/90)是模型能力的事,不是工程或数据能翻过去的。诚实的承诺是"S源稳71-72",不是"稳95"。
