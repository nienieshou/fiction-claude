# 复写成书命名 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把复写成书文件改成干净交付名 `<源ID><新书名>.txt`，可交付汇聚 `_deliverable/`、不可交付隔离 `_rejected/`。

**Architecture:** 仅改 `src/hiki/produce.py`。`_book_filename` 重写为两参纯函数；新增纯函数 `_delivery_path` 负责目录分流；`_stage_finalize` 改用甲格式正文头 + 两个纯函数接线。追溯信息（档/日期/状态）从文件名剥离，留在 `report.json`。

**Tech Stack:** Python 3.10+，pytest（`pythonpath=["src"]`，`from hiki import ...`）。

## Global Constraints

- 交付名格式：`<源ID><新书名>.txt`，源ID 直接拼接书名，无下划线、无《》、无档/日期/状态后缀
- 源ID 由 `_source_id(out_dir.name)` 抽取；书名经 `_safe_filename` 清 Windows 非法字符（半角 `:` 清除、全角 `：` 保留）
- 可交付落 `out_dir.parent / "_deliverable"`；不可交付落 `out_dir / "_rejected"`
- `.txt` 内容（甲）：首行 `《书名》`，空行，正文；无 title 退化为纯正文；不含 `#`/`>`/`---`
- `final.md` 仍写入 `out_dir`（原始正文，无头部），保持 web/诊断不受影响
- 所有文件 `encoding="utf-8"`
- 测试不烧改写钱：只测纯函数，不调 LLM

```python
def _delivery_path(out_dir: Path, deliverable: bool, out_name: str) -> Path:
    """交付件落盘路径：可交付汇聚 <out_dir 上级>/_deliverable/；不可交付隔离 <out_dir>/_rejected/。
    靠位置区分交付资格，文件名本身不带状态(A2-a)。"""
    base = (out_dir.parent / "_deliverable") if deliverable else (out_dir / "_rejected")
    return base / out_name
```
