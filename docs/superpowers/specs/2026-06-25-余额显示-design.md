# 左上角 DeepSeek 余额显示 设计

日期：2026-06-25
状态：已批准设计，待 writing-plans

## 背景

本会话刚修了 402 Insufficient Balance 崩溃（余额耗尽 → 任务批量崩、被误显示可续跑）。把 DeepSeek 余额**常驻显示在仪表盘左上角**，让操作者在任务崩溃前就看到余额低，是该根因的"可见化"。

DeepSeek 余额 API 契约（[doc](https://api-docs.deepseek.com/api/get-user-balance)）：
```
GET https://api.deepseek.com/user/balance   Header: Authorization: Bearer <DEEPSEEK_API_KEY>
→ {"is_available": bool, "balance_infos": [{"currency": "CNY"|"USD",
     "total_balance": "110.00", "granted_balance": "10.00", "topped_up_balance": "100.00"}]}
```

## 架构

- **后端代理**（key 在服务器 `.env`，浏览器不能直连 DeepSeek——会暴露 key + CORS）：新增 `GET /api/balance`，服务器带 key 调 DeepSeek，归一后吐前端。
- **前端**：`web/frontend/index.html` 的 `.brand` 块（index.html:104）保留 logo + `HIKI · 产线监视台`，把 `fiction-rewrite · pilot` 那行替换为**余额行**。

## 后端 `/api/balance`

新增 `web/backend/balance.py`：

- **纯归一函数** `normalize_balance(raw: dict) -> dict`：DeepSeek 原始 JSON → 前端契约。取 `balance_infos[0]`（首币种）。
  - 成功：`{"available": bool(raw["is_available"]), "currency": "CNY", "total": "110.00", "source": "live"}`
  - 原始 JSON 结构异常（无 balance_infos 等）→ `{"available": null, "total": null, "currency": null, "source": "error", "detail": "<原因>"}`
- **取数函数** `fetch_balance() -> dict`（async）：
  - 读 key：`os.environ.get("DEEPSEEK_API_KEY")`（确保 `load_dotenv(paths.ROOT/".env")` 已执行）
  - 无 key → `{"available": null, "total": null, "currency": null, "source": "no-key"}`（不调外网）
  - 有 key → `httpx.AsyncClient` GET `https://api.deepseek.com/user/balance`，`Authorization: Bearer <key>`，`timeout=5`
    - 2xx → `normalize_balance(resp.json())`
    - 非 2xx / 超时 / 连接异常 → `{"available": null, "total": null, "currency": null, "source": "error", "detail": "<状态码或异常类型>"}`
- **路由**（`web/backend/app.py`）：`@app.get("/api/balance")` → `await balance.fetch_balance()`。**绝不抛 → 不 500 仪表盘**（fetch_balance 内部已兜底所有异常）。

`httpx` 已在 `requirements.txt`（CI 修复时加入）。

## 前端余额行（`web/frontend/index.html`）

替换 `.brand` 块里 `<br><span>fiction-rewrite · pilot</span>` 为余额行：`<span id="balrow">…</span>`。

- **加载渲染 + 拉数**：页面初始化时调 `refreshBalance()`，渲染到 `#balrow`
- **节奏**：加载拉一次 → `setInterval` 每 **60s** 自动刷 → 余额行内 `↻` 按钮 `onclick=refreshBalance()` 手动刷
- **显示**：
  - 有数：`余额 ¥110.00 ↻`（CNY→`¥`、USD→`$`、其它→币种码前缀）
  - 无 key / error：`余额 — ↻`，元素 `title` 悬浮显示 `source`/`detail` 原因
- **告警分级**（阈值按 CNY；单本改写 ~¥8）。注意 `total` 是**字符串**（`"110.00"`），比较前必须 `parseFloat`（否则 `"9.00" < "10"` 字典序判错）：
  - `source` 非 live 或无数 → 灰（中性）
  - `is_available=false` 或 `parseFloat(total) < 10` → 红
  - `parseFloat(total) < 30` → 琥珀
  - 否则 → 中性/绿
- `refreshBalance()`：`fetch(API+"/api/balance")` → 失败（网络/后端 down）也不抛，显示 `余额 —`

## 测试

`tests/test_balance.py`（纯函数 + 取数，mock httpx，**零真实调用、零花钱**）：
1. `normalize_balance` 正常 JSON → `{available, currency:"CNY", total:"110.00", source:"live"}`
2. `normalize_balance` 结构异常（空 dict / 无 balance_infos）→ `source:"error"`
3. `fetch_balance` 无 key（monkeypatch.delenv）→ `source:"no-key"`，且**不发起 httpx 请求**
4. `fetch_balance` 有 key + mock httpx 2xx → `source:"live"`，total 正确
5. `fetch_balance` 有 key + mock httpx 抛/非2xx → `source:"error"`，不抛异常

前端单文件无单测：起后端冒烟，肉眼确认余额行渲染 + 告警色 + ↻ 刷新。

## 非目标（YAGNI）

- 不显示 granted/topped-up 明细（只总额）
- 不做服务器端缓存（单用户 pilot，60s poll 够）
- 不改既有 budget cap / 成本 ledger
- 不在余额低时自动暂停任务（仅显示告警；自动闸另案）
- 告警阈值（¥10/¥30）硬编码于前端，暂不做可配置
