# 复写成书命名（交付件）设计

日期：2026-06-24
状态：已批准设计，待 writing-plans

## 背景与问题

复写产出的成书文件，当前由 `produce.py:_book_filename` 命名为：

```
<源ID>_<档>_<YYYYMMDD>_《书名》[.不可交付].md
例：ZYGGY02252_A_20260624_《归隐田园：执子手共白头》.md
```

这是"科学命名"——为内部可追溯/可排序/状态显式服务。但它**不是交付名**：把档、日期、状态、书名号全堆进文件名，给编辑/平台时噪声大。

期望的交付名（用户给定）：源ID 继承自源、原长标题换成新商业书名、其余装饰全剥掉。

| 源文件名（带原标题） | 成品交付名（带新书名） |
|---|---|
| `ZYGGY02252穿成萌娃去流放？不怕，我把皇宫抄空了！.txt` | `ZYGGY02252归隐田园：执子手共白头.txt` |
| `ZYGGY02079买来的禁欲夫君是太子.txt` | `ZYGGY02079农女为后：皇上独宠我.txt` |

## 关键事实（决定低风险）

成书文件名是 **write-only 交付件**：web 控制台与所有下游读的是 `final.md` + `report.json` + `grade.json`，**没有任何程序解析成书文件名**。因此改名/改格式/换位置不影响仪表盘与对账。

## 决策

1. **命名格式**：`<源ID><新商业书名>.txt`
   - 源ID 由 `_source_id(out_dir.name)` 抽取（如 `ZYGGY02252`），**直接拼接**书名，无下划线、无《》
   - 书名 = `gen_title` 起的新商业书名，经 `_safe_filename` 清 Windows 非法字符（半角 `:` 被清、全角 `：` 合法保留）
   - 丢弃 档 / 生产日期 / `_full`·`_reval` 后缀；扩展名 `.txt`

2. **落盘位置（A2-a：全局汇聚 + 位置隔离）**
   - 可交付 → `output/_deliverable/<源ID><新书名>.txt`（全局交付目录，1 万本一处捞货）
   - 不可交付 → `output/<slug>/_rejected/<源ID><新书名>.txt`（靠位置隔离，绝不进交付目录）
   - `output/<slug>/final.md`、`report.json` 等原产物**原地不动**

3. **追溯信息（A：剥离但留档）**
   - 档 / 日期 / 交付状态 从文件名剥离，仍记在 `report.json`
   - `report.output_file` 字段更新为新交付文件的路径（含位置）
   - 源ID 为对账锚：交付名 → report.json → 源，链路不断

4. **`.txt` 内容格式（甲：纯文本头）**
   ```
   《书名》

   <正文>
   ```
   首行 `《书名》`，空行，正文。去掉 `#`/`>`/`---` 等 markdown 记号（.txt 里是噪声）。无 title 时（理论不应发生，`gen_title` 有兜底）退化为纯正文。

## 改动范围（最小）

仅 `src/hiki/produce.py`：

- `_book_filename(source_ref, safe_title)` → `str`：签名简化为两参，返回 `f"{_source_id(source_ref)}{safe_title}.txt"`。原 `grade`/`date`/`deliverable` 参数从命名职责中移除（状态/分流的判断上移到 finalize）。保留 `_source_id`、`_safe_filename` 复用。
- finalize 阶段（`_stage_finalize`，约 produce.py:1210–1240）：
  - 成书正文头改为甲格式：`f"《{title}》\n\n{final}"`（title 为空则 `final`）
  - **交付根** 取 `out_dir.parent`（batch 时 = `<out>`，单本时 = `output/`），由此推导：
    - 可交付 → `out_dir.parent / "_deliverable" / out_name`
    - 不可交付 → `out_dir / "_rejected" / out_name`
  - 目标目录 `mkdir(parents=True, exist_ok=True)` 后写入
  - `report["output_file"]` 更新为交付文件路径
  - `final.md` 仍照常写入 `out_dir`（保持原始正文，无头部）
- 废弃 `.不可交付` 文件名后缀逻辑（状态改由位置 + report 表达）

## 边界与一致性

- **重名**：源ID 全局唯一 → `<源ID><书名>` 唯一；best-of 重掷同一本只会覆盖同名文件（源ID 不变），符合预期
- **全局目录位置**：交付目录 = `out_dir.parent / "_deliverable"`，与各本 `out_dir` 同级。batch 多本共享同一 `<out>` 父目录 → 自然汇聚一处；单本 `output/<stem>_full` 的 parent 为 `output/` → 落 `output/_deliverable/`。无需新增传参
- **不可交付仍生成交付件**：A2-a 下不可交付也起干净名，仅落 `_rejected/`，便于诊断时直接读成书，但物理上隔离于交付目录
- **测试**：现有 `tests/` 不依赖成书文件名；新增针对 `_book_filename` 新格式 + 落盘分流的单测（纯函数 + 目录分流），不烧改写钱

## 非目标（YAGNI）

- 不改 `gen_title` 的书名质量/卖点逻辑
- 不改 slug / 输出目录命名规则
- 不为交付目录做软链/双写（A2-a 单写，非 A2-c）
- 不改 web 控制台（它读 final.md/report.json，无需动）
