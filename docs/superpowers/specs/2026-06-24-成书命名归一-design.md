# 成书命名归一（normalize）设计

日期：2026-06-24
状态：已批准设计，待 writing-plans
分支：并入 `feat/成书命名`（与命名改造同一 PR，处理存量为收尾）

## 背景与问题

[2026-06-24-成书命名-design.md](2026-06-24-成书命名-design.md) 把成书交付名改成 `<源ID><新书名>.txt`、可交付汇聚 `_deliverable/`。但**新命名上线前已产出的旧书**仍是旧文件，例如：

```
output/ZYGGY02079买来_reval/ZYGGY02079_A_20260623_《农女为后：皇上独宠我》.md
output/ZYGGY02252穿成萌娃_reval/ZYGGY02252_A_20260623_《归隐田园：执子手共白头》.md
```

它们的 `report.json` 已含归一所需的全部数据（`title`、`deliverable: true`，源ID 在目录名里），但文件名/位置仍是旧的、`output_file` 仍指向旧 `.md`。

**目标**：零成本把这些存量旧书迁移到新规范——读 `report.json` + `final.md` 重写出 `_deliverable/<源ID><新书名>.txt`，不重跑、不调 LLM、不花 ¥。

## 决策

1. **方式：纯重命名/迁移**（非重跑、非只改字段）。正文来源 = `out_dir/final.md`（已是干净正文）+ `report["title"]`，拼甲格式，与正常 `_stage_finalize` 同源。
2. **触发：CLI 命令为核心 + web 启动调用**（一套逻辑两入口）。
3. **分流**：按 `report["deliverable"]`——true → `_deliverable/`，false → `<slug>/_rejected/`（复用 `_delivery_path`）。
4. **幂等**：已归一的本（新 `.txt` 已存在且 `output_file` 已是新路径）跳过。
5. **旧 `.md` 删除**：迁移语义，归一后删掉旧科学命名文件；`final.md` 保留（诊断用）。

## 架构

### 新模块 `src/hiki/normalize.py`

不反向依赖 `web/`；复用 `produce` 的纯函数 `_book_filename` / `_delivery_path` / `_safe_filename`。

**`normalize_book(out_dir: Path, dry_run: bool = False) -> dict`** —— 归一单本：

1. 读 `out_dir/report.json`；不存在或无法解析 → 返回 `{slug, status: "skip-no-report"}`
2. `title = report.get("title") or out_dir.name`；`deliverable = bool(report.get("deliverable"))`
3. `final.md` 不存在 → 返回 `{slug, status: "skip-no-final"}`
4. `out_name = _book_filename(out_dir.name, _safe_filename(title))`；`new_path = _delivery_path(out_dir, deliverable, out_name)`
5. **幂等**：`new_path.exists()` 且 `report.get("output_file") == str(new_path)` → `{slug, status: "already", path: str(new_path)}`
6. 否则，`dry_run` → `{slug, status: "would-normalize", path: str(new_path)}`（不动盘）
7. 否则执行：
   - `new_path.parent.mkdir(parents=True, exist_ok=True)`
   - `body = f"《{title}》\n\n{final_text}"`（`final_text` 读自 `final.md`；`title` 为空时退化纯正文，但第 2 步已兜底为目录名，故恒有 title）
   - `new_path.write_text(body, encoding="utf-8")`
   - **删旧**：`old = report.get("output_file")`；仅当 `old` 为真、`(out_dir / old)` 存在、且 `(out_dir / old) != new_path` 时 `unlink()`。只认 `output_file` 明确指向的文件，绝不按模式猜测删除。
   - `report["output_file"] = str(new_path)`；写回 `report.json`（`ensure_ascii=False, indent=2`）
   - 返回 `{slug, status: "normalized", path: str(new_path)}`

**`normalize_tree(output_root: Path, dry_run: bool = False) -> list[dict]`** —— 遍历 `output_root` 的**直接子目录**（`iterdir()`，只扫一层），跳过非目录，逐本调 `normalize_book`，收集每本结果 dict。存量书均落在 `output/<slug>/` 直接子目录，一层足够；若有嵌套批量目录（如 `output/funnel/<slug>/`），把该父目录当 root 再跑一次即可（CLI 的 `root` 参数支持）。

### CLI 入口（`src/hiki/__main__.py`）

新增子命令：`hiki normalize [root] [--dry-run]`
- `root` 默认 `output`
- 调 `normalize_tree(Path(root), dry_run)`，打印每本 `status`，末尾汇总各状态计数（normalized / already / skip-* / would-normalize）

### web 入口（`web/backend/app.py` lifespan）

在现有 `lifespan`（app.py:52）里、`_auto_resume_stalled()` 旁加一次归一调用：
- `if os.environ.get("HIKI_WEB_NORMALIZE", "1") != "0":` → `from hiki import normalize; normalize.normalize_tree(paths.OUTPUT)`
- 默认开，仿 `HIKI_WEB_AUTORESUME`；异常不崩后端（try/except 包裹，打印告警）

## 边界与一致性

- 缺 `title` → `out_dir.name` 兜底（与 `point_repair` 一致）
- 缺 `final.md` → 跳过并记 status，不抛错
- `deliverable=false` → 落 `<slug>/_rejected/`，与 finalize 同构
- 删旧只认 `output_file` 字段，避免误删 `final.md` 或其它文件
- 幂等可重入：CLI 可反复跑、web 每次启动跑，已归一的本只读不写
- `output_root` 子目录里的 `batch_summary.json` 等非目录条目、以及 `_deliverable/` 自身（无 `report.json`）→ 自然跳过（`skip-no-report`）

## 测试（`tests/test_normalize_units.py`，纯文件级，零 LLM）

用 `tmp_path` 造假 out_dir：写 `report.json`（含 title/deliverable/旧 output_file）+ `final.md` + 旧 `.md`。断言：

1. **normalized**：调用后新 `.txt` 存在于 `_deliverable/`、内容为甲格式（`《title》\n\n` 开头，无 `#`/`>`/`---`）、旧 `.md` 被删、`report["output_file"]` 更新为新路径
2. **幂等**：第二次调用 status=`already`，不重复写
3. **skip-no-final**：删掉 final.md → status=`skip-no-final`，不动盘
4. **rejected 分流**：`deliverable=false` → 新文件落 `<out_dir>/_rejected/`
5. **dry-run**：status=`would-normalize`，磁盘无变化
6. **全角冒号/源ID**：title 含 `：`、out_dir 名带源ID → 新名 `<源ID><title>.txt` 正确、冒号保留

## 非目标（YAGNI）

- 不重跑改写、不重新起书名（`gen_title` 不调用）
- 不迁移 `final.md`/`bible.json` 等中间产物（只动成书交付件）
- 不做软链/双写
- 不改 `_deliverable/` 内已是新命名的本（幂等跳过）
