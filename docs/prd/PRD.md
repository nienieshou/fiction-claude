# hiki-fiction-rewrite · 产品需求文档（PRD v1.0）

**日期：** 2026-06-08
**关系：** 本 PRD 定义"做什么 / 成功是什么"（需求 + 验收）；架构实现见 `../design/system_design_final.md`（Final v5.0）。需求编号可追溯。

---

## 1. 产品概述

一个**全自动**中文网文**复写/压缩引擎**：批量输入任意章数、任意质量的网文（48–85 万字），产出**固定 60 章 / ~21 万字**的成品，使**网文读者人工评分稳定达 95**。仅用 deepseek-v4，生产链路零人工，单本 ≤¥50，规模化处理 1万本+。

**一句话价值**：把"高分但注水/有瑕疵的长篇网文"提纯为"更短、更密、更好看、可商用"的精品，并能对鱼龙混杂的海量源做到"好的高保真提纯、弱的升级、救不动的拒收"。

---

## 2. 问题与目标

- **问题**：旧系统（参考源从零创作）人工仅 60–70，自动自评 93–96（Goodhart 虚高 28–31 分），靠堆硬规则边际递减。
- **根因**：抛弃源已验证的质量、漂文艺腔、丢钩子人物、压缩成流水账、自评当目标。
- **目标**：以"源是质量脊柱 + 提分靠选不靠写 + 评估靠对照破 Goodhart + 人作老师离线校准"，**稳定产出网文读者 95 分**的成品，全自动、可规模化、可维护。

---

## 3. 范围

**In scope**：源清洗 → 准入分级 → 故事 DNA 提取 → 60 章规划 → 场景复写 → 质量门禁 → 升级/拒收 → 组装 → 人工反馈学习闭环；1万本批量、断点续跑、成本/可观测。
**Out of scope**：从零原创小说；插画/有声书/排版出版；实时交互式写作；非中文；人在生产链路内的编辑。

---

## 4. 用户与场景

- **运营方（主用户）**：投入海量源 txt，期望自动批产出货达标成品，只在**事后逐本打分点评**（老师角色）。
- **人工评审员**：按统一 rubric 给成品打分+点评，喂学习飞轮（不进生产链路）。
- **典型场景**：万本级流式批处理；单本失败隔离；难书多烧算力、易书少烧、救不动拒收。

---

## 5. 约束（全部锁定）

| # | 约束 |
|---|---|
| C1 | 仅 **deepseek-v4**（v4-pro / v4-flash，1M 上下文，缓存近免费）；无 Claude/GPT/Gemini |
| C2 | **生产链路零人工**；人=老师（事后逐本打分点评） |
| C3 | 单本成本 **≤¥50**（自适应；评估为成本大头） |
| C4 | 输出 **固定 60 章 / ~21 万字**（每章 ~3500） |
| C5 | **拒收率 ≤50%**；救不动/不可压缩的源拒收 |
| C6 | **原创性下限**：保内核、换皮重写表达、禁逐字照搬（避版权+去 AI 检测） |
| C7 | 评分者 = **网文读者**；"95"=网文标准顶级好看，**非文学性** |
| C8 | 尽量高并发压缩生产周期；顶级泛化（各类题材） |

**输入现实**：标杆语料 = `codex/fictions_source/` **40 本人工 90 分**（高档锚定/参考种子）；生产输入 = 1万本鱼龙混杂；脏数据多（防盗章/广告/作者的话/乱码/多书拼接）。

## 6. 质量定义："95"是什么（本 PRD 的心脏）

**95 = 双轴最弱环（章 × 维度）全部达标 + 整本涌现"好看"**。不是加权平均（平均会藏致命弱维）。

**6–8 个正交维度，两层：**
- **场景级（驱动修复）**：钩子张力 / 爽点兑现 / 人物魅力与代入 / 对话鲜活 / 画面感 / 语言净度（去 AI 腔+语病+原创性）
- **整本级（驱动 holistic 认证）**：弧光收束 / 节奏曲线 / 情感总冲击 / 主题统一 / 结尾落点 / 全书连续性

**5 个可量化结构指标（0-LLM、确定性、不可刷分；从逐章摘要/DNA 计算）**：
| 指标 | 计量 |
|---|---|
| 金手指释放节奏 | 能力解锁章号分布（分段 vs 前置；条件指标） |
| 爽点密度/间隔 | 平均爽点间隔 + **最大憋屈跨度**（弃书杀手） |
| 钩子密度 | 平均钩子间隔 + 大钩子悬置跨度 |
| 阶段结构均衡度 | 各卷章数占比 + 水段（事件/爽点/钩子三低章） |
| 结局收束完整度 | **伏笔闭环率** + 末段仓促度 |

- 结构指标 = **必要地板 + 早警**（不充分）；**0-100 评分曲线按题材校准**，不硬编码。
- 防刷分：**摘要↔正文一致性校验**（摘要标爽点，正文须实测兑现）。

## 7. 功能需求（FR）

| # | 需求 |
|---|---|
| FR-1 摄入清洗 | 解析任意 txt；去防盗章/广告/作者的话、编码修复、章节完整性、去重、**多书拼接边界检测**；产 `clean.txt`+质检指标 |
| FR-2 准入分级 | 按**质量 S/A/B/C/D/Q × 可压缩性**（form-incompatible→Q）分级；Q 不进付费阶段 |
| FR-3 DNA 提取 | 抽脊柱/钩子账/情感曲线/人物弧/伏笔/爽点/人名词典/语域指纹/题材；1M 上下文大块入栈+缓存源 |
| FR-4 60 章规划 | 分层（全书弧→卷→章→场景）+ 一致性回环；**Plan 阶段即满足 §6 结构指标**（shift-left）；自带压缩保真校验 |
| FR-5 场景复写 | 以场景为原子单元；DRAMATIZE/SUMMARIZE；"提纯源"非"重写"；锁源语域、删五类注水、修语病、人名归一 |
| FR-6 升级阶梯 | 模式1 保真压缩→2 强化改写→3 类型化重构→4 概念级重启；准入定入口，闸门不过下沉，最深仍不过即拒；C/D 注入参考水库 |
| FR-7 选优精修 | 场景级生成 N 候选→Elo 锦标赛选优→对最弱维度精修（带**反回归门**）；戏剧高点造峰；选择性大 N（按预算） |
| FR-8 质量门禁 | 机械层(含结构指标)一票否决 + 双锚对照 PK(金标95/标杆90/源) + 整本 holistic + 维度 floor；详见 §8 |
| FR-9 组装交付 | 拼 60 章 → `final.md` + 验收报告（认证/成本/风险）；含标题/简介 |
| FR-10 反馈学习 | 人工逐本打分点评入 HFL；训偏差校准器、维度偏差图、点评挖掘扩检测器、反哺金标/参考库 |
| FR-11 拒收隔离 | 触成本上限/最深模式不过/不可压缩 → 标记隔离，不出货 |

## 8. 质量门禁与验收（破 Goodhart 治理）

- **认证 95** = 机械层(含结构指标 floor)过 ⊕ 每场景维 floor 过 ⊕ 每整本维 floor 过 ⊕ holistic PK 达金标(95)。
- **冷启动**靠外部金标(95)+40 本标杆(90)锚；**稳态**由人工逐本打分持续校准（把自动虚高显式拟合掉）。
- **评分治理原则（最高纪律，防 Goodhart）**：
  1. **评分是指南针非自动驾驶**——靠**选**（锦标赛）不靠**拧**（禁直接优化奖励模型分）。
  2. **分要活不要冻**——持续被新鲜人工真值 + 外部金标重锚。
  3. **便宜量化分只做地板**——结构指标话语权封顶；**主观维度 PK + 整本 holistic + 人工真值为认证主导**（防"结构完美、灵魂死掉"）。

## 9. 非功能需求（NFR）——简洁优雅 / Reliable / Scalable / Maintainable

**NFR-Elegant（简洁优雅）**
| # | 要求 |
|---|---|
| E1 | 架构由 8 条设计公理推导；核心仅 5 个正交概念（管线/场景/质量环/闸门/飞轮）。新增物须归入某公理 |
| E2 | 一个质量环服务全部模式；引擎=模式内动作，不另立概念 |

**NFR-Reliable**
| # | 要求 |
|---|---|
| R1 | 文件即状态；阶段纯函数、幂等、可独立 rerun；全量断点续跑 |
| R2 | 所有 LLM 输出 schema 校验，坏数据拒收不落盘 |
| R3 | 锦标赛非确定 → 候选/PK 结果缓存+种子化，崩溃不重烧、可复现 |
| R4 | 单本预算硬上限 ¥50，跑飞 abort 转拒收；单本失败隔离，不拖累全批 |
| R5 | LLM 失败退避重试，仍失败转隔离队列（优雅降级） |

**NFR-Scalable**
| # | 要求 |
|---|---|
| S1 | 两级并发：inter-book 工作队列+无状态 worker 池；intra-book 场景扇出 |
| S2 | 缓存源上下文（同源被 60 章×锦标赛×评估反复复用）为省钱/提速命脉 |
| S3 | 限流感知客户端+指数退避；成本随难度自适应（非一刀切） |
| S4 | 学习飞轮异步解耦于生产，互不阻塞；支撑万本流式 |

**NFR-Maintainable**
| # | 要求 |
|---|---|
| M1 | 凡影响产出者皆**版本化资产**（prompt/闸门/金标/检测器/偏差模型/config）；升级对金标库做回归测试不得退化 |
| M2 | 配置驱动（模式/预算/阈值/路由在 config，不在代码）；prompt 为数据模板 |
| M3 | 阶段间契约=schema；可观测内建（通过率/拒收率/false-reject/认证↔人工误差/PK胜率/成本/覆盖度/漂移告警） |

**NFR-Cost**：单本 ≤¥50（强源走模式1 偏下沿）；评估为成本大头，N/PK 按预算自适应。

- **标杆语料**：`codex/fictions_source/` 40 本（人工 90）→ anchors_90 + 参考水库种子 + 闸门近目标锚。
- **全局版本化资产**：`assets/{prompts,gate,gold_corpus(95固定),anchors_90,ref_reservoir(滚动),detectors,bias_model,config}`。

- 模型路由：Extract/Plan/整本闸门/终裁 PK = v4-pro；Draft/初筛/机械 = v4-flash。

## 13. 风险与开放问题

1. **强源 90→95 的 +5 拔高**（已大降级）：纯 DeepSeek、¥50、标杆强源到 95——M0 先验；难点真正在 10k 弱源（靠模式升级+拒收兜底）。
2. **【命门】闸门判别力**：DeepSeek 区分 90↔95 有上限；双锚+人工持续校准缓解，天花板靠微调突破。
3. **压缩 vs 网文爽感**：4:1 压缩或伤养成/追更感 → 可压缩性维度兜底。
4. **学习速度 < 生成速度**：放量前置闸防裸奔出万本未校准货。
5. **微调路线**：DeepSeek 无官方微调 API；V4 开源权重(MIT)可自托管微调——后期最大杠杆，重运维、非快赢。
6. **A1/A2 二选一（待定）**：成品与源的差异化目的（同故事更高质量 vs 可独立发表）——影响是否加差异化映射层。

## 14. 依赖与假设

- DeepSeek-v4 API 稳定可用、1M 上下文与缓存计价如核查；限流满足万本吞吐。
- 人工评审按统一 rubric、有 IRR 控制；逐本打分速度 M0–M3 可支撑（M4 转快评+抽样）。
- 40 本标杆人工分 90 为可信基准。
